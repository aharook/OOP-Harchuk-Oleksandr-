## Тема: 
Юніт-тестування: принципи, фреймворки, моки.


## 1. Юніт-тестування: теорія та практика

Юніт-тестування (unit testing) — це підхід до перевірки програмного забезпечення, при якому тестуються найменші незалежні частини програми. 
Під час юніт-тестування створюються невеликі автоматизовані тести, які викликають конкретний метод або функцію з певними вхідними даними та перевіряють, чи відповідає результат очікуваному значенню. Якщо результат не збігається з очікуванням, тест вважається проваленим, що сигналізує про наявність помилки.
Юніт-тестування зазвичай використовує спеціальні фреймворки тестування. Для C++ популярними є Google Test, Catch2 або doctest. Такі інструменти дозволяють створювати тести, порівнювати результати та формувати звіти про виконання.

## 2. Переваги та обмеження юніт-тестування порівняно з інтеграційним тестуванням

Юніт-тестування має багато переваг порівняно з інтеграційним тестуванням. 
Насамперед воно дозволяє дуже точно визначити місце виникнення помилки. Оскільки тест перевіряє лише один компонент, у разі збою стає зрозуміло, який саме метод або функція працює неправильно. Крім того, такі тести виконуються дуже швидко. Ще однією перевагою є можливість перевіряти код на ранніх етапах розробки. 

Однак юніт-тестування має і певні проблеми. 
Воно перевіряє лише окремі частини програми і не показує, як ці частини взаємодіють між собою. 
Тому, для роботи зі зв'язками використовується інтеграційне тестування. Такі тести можуть виявити проблеми сумісності, неправильний обмін даними або логічні помилки у взаємодії компонентів. Недоліком інтеграційного тестування є те, що воно складніше у реалізації та виконується повільніше.

## 3. Приклад класу та юніт-тести
Створено клас Calculator, який виконує базові арифметичні операції. У ньому реалізовано три методи: додавання, віднімання та ділення.

### Клас Calculator

    class Calculator {
    public:
        int Add(int a, int b) {
            return a + b;
        }

        int Subtract(int a, int b) {
            return a - b;
        }

        double Divide(double a, double b) {
            if (b == 0) {
                throw std::invalid_argument("division by zero");
            }
            return a / b;
        }
    };

### Юніт-тести для класу

    void TestAddSuccess() {
        Calculator calc;
        assert(calc.Add(2, 3) == 5);
    }

    void TestAddBoundary() {
        Calculator calc;
        assert(calc.Add(0, 0) == 0);
    }

    void TestSubtractSuccess() {
        Calculator calc;
        assert(calc.Subtract(10, 4) == 6);
    }

    void TestSubtractBoundary() {
        Calculator calc;
        assert(calc.Subtract(5, 5) == 0);
    }

    void TestDivideSuccess() {
        Calculator calc;
        assert(calc.Divide(10, 2) == 5);
    }

    void TestDivideBoundary() {
        Calculator calc;
        bool exceptionThrown = false;

        try {
            calc.Divide(10, 0);
        }
        catch (const std::invalid_argument&) {
            exceptionThrown = true;
        }

        assert(exceptionThrown);
    }

## 4. Використання mock-об’єктів

Mock-об’єкти використовуються у випадках, коли тестований клас залежить від інших компонентів системи. У таких ситуаціях створюється спрощена імітація, яка повертає контрольовані значення.

Проте mock-об’єкти не завжди необхідні. Якщо клас не має складних зовнішніх залежностей і використовує лише прості обчислення або внутрішні дані, тестування можна проводити без них. У такому випадку достатньо створити реальний об’єкт класу та викликати його методи.
